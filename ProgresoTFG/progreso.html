Actualizado a 2026-06-18

1. Despliegue base del laboratorio

Hecho

Se documentó la estructura inicial con MariaDB, la app moderna y la app clásica, además de la red interna para aislar la base de datos.

2. Corrección de errores de arranque

Hecho

Se resolvieron el fallo de arquitectura de DVWA en ARM64, el problema del package.json en la app moderna y el conflicto de dependencias de Next.js.

3. Publicación perimetral con Apache

Hecho

Se crearon los subdominios, se definió la IP permitida con allowed_ip.conf, se configuraron los VirtualHost y se evitaron fugas de información por redirecciones globales.

4. Script para cambiar la IP autorizada

Hecho

Se añadió update_ip.py para actualizar la IP pública del atacante, validar la sintaxis de Apache y recargar el servicio sin cortar conexiones.

5. Pruebas de acceso perimetral

Hecho

Se verificó que solo la IP autorizada accede a los subdominios y que el resto de peticiones recibe 403 Forbidden.

6. Instalación del agente Wazuh

Hecho

Se registró la instalación nativa del agente, la configuración del manager y la comprobación del estado del servicio y de la sincronización con el servidor.
