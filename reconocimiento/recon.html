Estrategia Táctica: Centralización en Kali Linux

Para mantener una higiene metodológica profesional y facilitar el posterior filtrado de eventos en el SIEM, se recomienda centralizar todas las herramientas ofensivas desde tu máquina Kali Linux:

  • Consistencia de IP: Al lanzar los ataques y escaneos de vulnerabilidades desde una sola IP en Kali, se simplifica el filtrado en el access.log de Nginx y en los paneles de Wazuh.
  • Gestión unificada: Se evita fragmentar la superficie de ataque entre sistemas Windows y Kali, manteniendo las trazas en un único nodo origen.

Ejecución de OWASP ZAP desde Kali (Entorno Remoto o Headless)

Si estás operando sobre una VM de Kali sin entorno de escritorio o deseas realizar los escaneos en segundo plano:

Opción A: Ejecución gráfica clásica

zaproxy

Opción B: Modo Daemon con API Web (Acceso remoto)

Lanza el escáner como servicio en la terminal de Kali y accede al asistente gráfico desde el navegador de tu máquina Windows principal:

Terminal Kali Linux (Headless) 
zap.sh -daemon -host 0.0.0.0 -port 8080 -config api.disablekey=true

Una vez ejecutado, accede en el host a: http://IP_DE_KALI:8080 para usar la API interactiva.

Flujo Recomendado de Pentesting

Para maximizar las evidencias y registros en el SOC, sigue esta secuencia lógica de ejecución:

  1. Fase 1: Reconocimiento: Ejecución de escaneos DNS y nmap.
  2. Fase 2: Escaneo DAST: Lanzar OWASP ZAP a los dos subdominios DuckDNS para registrar el tráfico en Wazuh.
  3. Fase 3: Explotación y Control: Explotar el panel de login/búsqueda con SQLMap y detonar la shell reversa por RCE.
