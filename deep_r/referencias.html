Fuentes Bibliográficas, Referencias y Manuales Web

Inventario exhaustivo de enlaces externos, portales oficiales, documentaciones técnicas y manuales de referencia de las aplicaciones desplegadas en el proyecto.

1. Gobernanza, Gestión de Riesgos y Regulaciones

Marcos oficiales y normativas del sector público y privado para gobernanza y análisis cuantitativo de riesgos:

  • Metodología de Análisis de Riesgos MAGERIT v3: Guía oficial de MAGERIT (PAE) y catálogos de salvaguardas.
  • Esquema Nacional de Seguridad (ENS) - CCN-CERT: Instrucciones Técnicas de Seguridad (ITS) e informes oficiales del ENS.
  • Guías CCN-STIC del Centro Criptológico Nacional: CCN-CERT - Guías de seguridad de sistemas y tecnologías de la información.
  • INCIBE (Políticas de Seguridad para PYME): INCIBE - Manuales de directrices y plantillas de políticas de seguridad aplicables a empresas.

2. Auditoría Ofensiva, Pentesting y Exploits

Enlaces a repositorios y manuales de las herramientas tácticas de explotación y análisis de intrusión:

  • Nmap Security Scanner (Manual de Usuario): Manual y documentación de comandos de descubrimiento de puertos de Nmap.
  • DNSdumpster: Herramienta en línea de investigación de registros DNS e inteligencia OSINT.
  • Damn Vulnerable Web Application (DVWA) - Guía UPM: Guía académica de explotación de DVWA (TFG Universidad Politécnica de Madrid).
  • Exploit RCE React2Shell (RCE Portal): Portal interactivo del detonador de payloads ofensivos RCE contra endpoints React2Shell.
  • Guía de Explotación React2Shell (Writeup CVE-2025-55182): Análisis de la vulnerabilidad y guía técnica del exploit en GitBook.
  • Defensa contra React2Shell (Microsoft Security): Análisis detallado de Microsoft Security sobre la defensa contra la vulnerabilidad CVE-2025-55182.
  • React Server Components Security (Blog Oficial): Comunicado oficial del equipo de React sobre la vulnerabilidad crítica en React Server Components.
  • DVNA (Damn Vulnerable NodeJS Application): Proyecto de aplicación vulnerable en Node.js por Appsecco.
  • Vulnerable Node.js App (Payatu): Entorno de pruebas y laboratorio vulnerable por Payatu.
  • Vulnerable Node.js/Express/MySQL (Stypr): Contenedor e infraestructura vulnerable por Stypr.
