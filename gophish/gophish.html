Simulación de Phishing Corporativo con Gophish

Guía técnica paso a paso para la instalación, configuración y ejecución de campañas de concienciación sobre ingeniería social. Esta documentación describe los procedimientos periciales para desplegar **Gophish**, configurar credenciales de administración, clonar plantillas reales y auditar las respuestas del factor humano.

Descarga e Instalación

Gophish está desarrollado en Go, lo que proporciona un binario único auto-contenido libre de dependencias complejas. La descarga oficial se realiza desde getgophish.com o directamente desde su repositorio de GitHub.

1. Obtención y extracción del software

Puedes descargar el binario precompilado para tu sistema operativo desde la sección oficial de lanzamientos. A continuación se detallan las instrucciones para su despliegue tanto en Windows 11 (entorno de pruebas local) como en Linux (Kali / Debian):

Despliegue en Windows 11 (Entorno Local)

1. Accede a las descargas de Gophish y obtén el paquete ZIP comprimido correspondiente a Windows (ej: gophish-v0.12.1-windows-64bit.zip).
2. Descomprime todo el contenido en una ruta de tu disco local (ej: C:\Proyectos\gophish\).
3. Abre una terminal de **PowerShell** o **Símbolo del sistema** con privilegios de **Administrador** y navega hasta el directorio de la aplicación. 

# Acceso y ejecución en PowerShell (Administrador) desde el Escritorio
cd $HOME\Desktop\gophish
.\gophish.exe
Despliegue en Linux (Kali / Debian de Auditoría)

1. Utiliza wget en tu consola para descargar el paquete de instalación ZIP de 64 bits para Linux.
2. Utiliza unzip para extraer el binario ELF en tu máquina y configúralo como ejecutable mediante chmod.
3. Ejecuta con privilegios de superusuario (sudo) para permitir la apertura de sockets en puertos bajos (puerto 80/443). 

# Comandos en Consola Linux
wget https://github.com/gophish/gophish/releases/download/v0.12.1/gophish-v0.12.1-linux-64bit.zip
unzip gophish-v0.12.1-linux-64bit.zip
chmod +x gophish
sudo ./gophish

2. Primer Lanzamiento y Obtención de Credenciales

Al arrancar Gophish por primera vez, el sistema genera de forma segura un **token/contraseña aleatoria de administración** y la imprime en la salida de consola (stdout). Es fundamental copiar esta clave para el primer inicio de sesión.

Arranque Inicial (Terminal) 
# Ejecutar la aplicación como superusuario (requerido para abrir el puerto 80)
sudo ./gophish

Configuración Inicial (config.json)

Por defecto, la interfaz de administración de Gophish solo es accesible localmente (`127.0.0.1:3330`). Para administrarlo remotamente o cambiar los puertos de escucha para que no entren en conflicto con otros servicios del laboratorio, editamos el archivo `config.json`. Cabe destacar que en Windows ningún puerto interfería con la ejecución de este servicio en el momento de realizar el laboratorio.

config.json (Configuración Estructurada) 
{
  "admin_listener": {
    "listen_url": "127.0.0.1:3333",
    "use_tls": true,
    "cert_path": "gophish.crt",
    "key_path": "gophish.key"
  },
  "phish_listener": {
    "listen_url": "0.0.0.0:80",
    "use_tls": false
  },
  "db_name": "sqlite3",
  "db_path": "gophish.db",
  "migrations_path": "db/db_sqlite3/migrations/"
}
Mantenimiento y Copias de Seguridad:
  • Base de Datos (`gophish.db`): Gophish guarda todo el historial en un único archivo SQLite. Para hacer copias de seguridad de la campaña, basta con respaldar este archivo en frío.
  • Certificados TLS (`gophish.crt` y `.key`): El framework auto-genera certificados de prueba SSL al primer arranque. Para producción, deben enlazarse a un certificado válido (ej. Let's Encrypt) para evitar la alerta de sitio peligroso en el navegador del administrador.

Arquitectura y Funcionamiento de la Simulación

La plataforma funciona estructurando un embudo de eventos que permite medir la concienciación de los usuarios. En este laboratorio de TFG, orientamos la campaña específicamente a simular la captura de credenciales del ecosistema iCloud. Con fines exclusivamente académicos y éticos, no se emplean logotipos oficiales de marcas registradas, sino que se recrea únicamente la estética y la disposición visual del formulario. Asimismo, para el envío de los correos utilizamos el canal legítimo de Gmail como servidor SMTP de salida, a diferencia de los atacantes reales que suelen registrar dominios cebo similares al legítimo (como icloud.com) para engañar al usuario. Cuando se despacha un email, Gophish inyecta identificadores dinámicos que diferencian las acciones de cada destinatario de manera individualizada:

Acción de Click Redirecciona por ID único

Configuración del Perfil de Envío SMTP

Para que los correos electrónicos se envíen de manera fiable, Gophish requiere configurarse con un servidor SMTP externo. Por razones de seguridad perimetral, Google exige el uso de una credencial dedicada de 16 caracteres, aislando la contraseña maestra del canal SMTP.

Parámetros del Perfil de Envío SMTP:

  • Name: Nombre identificativo (ej. Soporte IT Gmail Relay).
  • From: El remitente falso visible (ej. Soporte IT TFG <soporte.it.masterciber@gmail.com>).
  • Host: El servidor y puerto SMTP con TLS activo (ej. smtp.gmail.com:587).
  • Username & Password: Credenciales de autenticación. En el caso de Gmail, es obligatorio usar una Contraseña de Aplicación de 16 dígitos generada desde el panel de seguridad de Google.

Creación de Plantilla y Landing Page

El éxito pericial del phishing reside en la similitud visual del cebo. Gophish automatiza la generación y captura de estos recursos de forma estructurada.

1. Email Template (Diseño del Cebo con Estética iCloud)

Se estructuró un diseño corporativo limpio utilizando tablas HTML y estilos CSS en línea compatibles con clientes de correo modernos, inyectando la variable de escucha {{.URL}} y agregando el tag dinámico {{.Tracker}} al final del cuerpo del correo.

2. Landing Page (Interrupción y Captura Activa)

Se configuraron las directivas 'Capture Submitted Data' y 'Capture Passwords' en conjunto con el parámetro 'Redirect To' apuntando al despliegue remoto en Vercel.

Lanzamiento y Ejecución de la Campaña

Una vez configurados los perfiles SMTP, las plantillas y los portales de interceptación, agrupamos los objetivos de prueba agregando una lista en "Users & Groups" e iniciamos el envío controlado. La campaña vincula la IP privada del servidor (http://192.168.1.73) para habilitar la resolución del dispositivo móvil mediante la red Wi-Fi local.

Panel Analítico de Control y Resultados periciales

Gophish dispone de un panel principal dinámico en tiempo real que mide la evolución temporal del impacto pericial, dividiendo los resultados en un embudo secuencial de eventos:

Paso 1 Email Sent

Correos despachados satisfactoriamente por el agente SMTP.

Paso 2 Email Opened

El usuario abre el correo detonando la llamada oculta al píxel `track.gif`.

Paso 3 Clicked Link

El usuario pulsa el botón del email y es dirigido a la landing page perversa.

Paso 4 Submitted Data

El usuario introduce credenciales en la landing, capturándose en el panel central.

El sistema registró de forma secuencial la apertura, la interacción con el enlace y el posterior envío del formulario, almacenando de forma segura los strings de prueba ingresados por el usuario antes de enviarlo a la pantalla de concienciación.
